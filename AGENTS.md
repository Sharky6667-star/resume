# 通用个人介绍维护规则

## 项目定位

- 这是长期复用的个人介绍页，不绑定具体公司或岗位。
- 不加入照片、求职意向、公司品牌分析或其他定向文案。
- 三个项目、三段实习及量化数据必须有现有简历或复盘材料支持。

## 文件边界

- `index.html` 是网页内容与样式的唯一源文件。
- `黄炯鹏简历.pdf` 是唯一下载文件名；修改名称时必须同步 HTML、README 和测试。
- `tests/verify.js` 是发布前必跑的验收脚本。
- 预览图由测试生成并被 `.gitignore` 忽略，不提交到仓库。

## 交互红线

- 黑色“项目 / 实习 / 联系”导航必须在离开首页后吸顶。
- 桌面端保持三栏；移动端保持紧凑横向三栏，吸顶高度不得超过 96px。
- 项目与实习正文默认可见，不得依赖 JavaScript 动画后才显示。
- 章节必须保留顶部滚动距离，不能被吸顶导航遮挡。

## 发布流程

1. 在本地运行 `tests/verify.js`，确认桌面端和移动端均通过。
2. 只向 `main` 推送通过验证的内容。
3. 等待 GitHub Pages 部署后，实际打开 `https://sharky6667-star.github.io/resume/`。
4. 在线确认页面文字、吸顶导航及 `黄炯鹏简历.pdf` 下载链接正常后，才能报告发布完成。
